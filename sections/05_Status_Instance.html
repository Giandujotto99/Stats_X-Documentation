Overview

FStatusInstance is the runtime representation of an active status effect. It holds all persistent state needed across execution ticks: timing, stack count, saved program counter, memory blobs, object references, and actor pointers.

Instances are not created automatically — the VM executes bytecode inline using a transient global context, and only when a handler explicitly calls RegisterAsActiveInstance() is an instance allocated from the pool.

Source: Core/StatusInstance.h, Core/StatusInstance.cpp

Design Philosophy

Principle Implementation
Handler-driven creation Instances exist only when needed (e.g., a status has a delay, loop, or duration). Fire-and-forget statuses never allocate an instance
Cache-line optimization Hot tick data (64 bytes) fits in a single L1 cache line. Cold data starts at offset 64+
Pool-managed lifecycle Instances are allocated from FStatusInstancePool and recycled via Reset(), avoiding GC pressure
Separate from VM context The VM uses a single global FForgeVMContext. Instances store only data that must persist between execution slices

Cache-Line Optimized Layout

The struct is explicitly organized so that all fields accessed every tick fit within the first 64-byte cache line:

Structure 
┌─────────────────────────── Cache Line 1 (64 bytes) ────────────────────────────┐
│ Offset  Size  Field                                                             │
│ ──────  ────  ──────────────────────────────────────────────────────────────── │
│   0      8    StatusID (int64)                                                  │
│   8      1    State (EStatusInstanceState)                                      │
│   9      3    [padding]                                                         │
│  12      4    ElapsedTime (float)                                               │
│  16      4    TimeSinceLastTick (float)                                         │
│  20      4    Duration (float)                                                  │
│  24      4    BaseDuration (float)                                              │
│  28      4    TickInterval (float)                                              │
│  32      4    NextExecutionTime (float)                                         │
│  36      4    SuspendTimer (float)                                              │
│  40      4    MaxIterations (int32)                                             │
│  44      4    CurrentIterationCount (int32)                                     │
│  48      4    StackCount (int32)                                                │
│  52      4    SavedPC (int32)                                                   │
│  56      1    bWaitingForAsyncEvents (bool)                                     │
│  57      7    [padding to 64-byte boundary]                                     │
├─────────────────────────── Cache Line 2+ (cold data) ──────────────────────────┤
│  64+     ~    StatusTag, Definition, Actor/Component pointers, Blobs...         │
└─────────────────────────────────────────────────────────────────────────────────┘

Compile-time enforcement:

C++ 
static_assert(offsetof(FStatusInstance, StatusTag) >= 64,
    "FStatusInstance: Cold data must start at or after offset 64 (cache line boundary)");

Hot Data — Cache Line 1

Fields accessed by UpdateTiming(), ShouldTick(), and ShouldResume() every frame:

Field Type Default Description
StatusID int64 0 Unique instance identifier (generated by UStatsX_GameInstanceSubsystem)
State EStatusInstanceState Inactive Current lifecycle state
ElapsedTime float 0.0 Monotonic lifetime timer since registration (never resets)
TimeSinceLastTick float 0.0 Accumulator since last tick execution (reset on each tick)
Duration float -1.0 Absolute expiry point on ElapsedTime timeline. ≤ 0 = infinite
BaseDuration float -1.0 Original finite duration (used for deterministic refresh semantics)
TickInterval float 0.0 Seconds between ticks. 0 = every frame
NextExecutionTime float 0.0 Next scheduled execution time (for heap scheduling)
SuspendTimer float 0.0 Countdown timer for suspended state. Resumes when ≤ 0
MaxIterations int32 -1 Maximum tick iterations. -1 = infinite
CurrentIterationCount int32 0 How many ticks have executed so far
StackCount int32 1 Current number of stacks
SavedPC int32 0 Program Counter to resume from when unsuspended
bWaitingForAsyncEvents bool false True while waiting for async event dispatch (montage callbacks, etc.)

Cold Data — Cache Line 2+

Fields accessed only during initialization, resumption, or termination:

Field Type Description
StatusTag FGameplayTag Tag identifying this status type (copied from Definition)
Definition const UPDA_StatusDefinition* Pointer to the compiled bytecode asset
CasterActor TWeakObjectPtr<AActor> Actor that cast this status
TargetActor TWeakObjectPtr<AActor> Actor receiving this status
CasterComponent TWeakObjectPtr<UStatsX_StatsComponentBase> Cached caster component (resolved once at init)
TargetComponent TWeakObjectPtr<UStatsX_StatsComponentBase> Cached target component (resolved once at init)
Revision int32 Monotonic per-instance revision. Incremented on each emitted status update. Starts at 1

Why TWeakObjectPtr?

Actor and component references use TWeakObjectPtr because:

  • The status instance may outlive the actor (actor destroyed while status still in pool)
  • Weak pointers auto-null when the referenced object is garbage collected
  • The OwnerDestroyed termination reason handles cleanup when pointers become stale

Persistent Memory

InstanceBlob

C++ 
TArray<uint8> InstanceBlob;

Raw byte storage for persistent node state (Gate/DoOnce flags, async node data, Instance-scoped variables). Size is pre-calculated by the compiler (Definition->InstanceBlobSize).

Layout:

Structure 
┌─────────────────────────────────┬──────────────────────────────────┐
│  Gate/DoOnce Region             │  User Instance Variables         │
│  (GateStateRegionBytes)         │                                  │
└─────────────────────────────────┴──────────────────────────────────┘

InstanceObjectPool

C++ 
TArray<TObjectPtr<UObject>> InstanceObjectPool;

GC-safe object reference storage. The InstanceBlob stores uint16 indices into this pool instead of raw 64-bit pointers. This provides:

  • GC safety — UObject references are tracked by the collector via UPROPERTY
  • 16-bit compression — instruction payloads use 16-bit indices
  • Deduplication — AllocObject() performs linear scan dedup (optimal for typical 2–20 entry pools)

VariableBlob

C++ 
TArray<uint8> VariableBlob;

Dedicated storage for user-declared variables that persist across execution slices. Initialized from context at instance creation time with full object reference remapping.

ArrayPool

C++ 
TArray<FForgeArraySlot> ArrayPool;

Persistent dynamic array storage. Array variables store uint16 handles into this pool. Each slot tracks element type, count, and raw byte data with geometric growth.

Function Call State

When a status instance suspends while inside a function call (nested event blocks), the full call state is persisted:

Field Type Description
FunctionCallStack TArray<FForgeFunctionCallFrame> Call frames: return PC, output descriptor, argument range
FunctionArgSlots TArray<FForgeFunctionArgSlot> Metadata for each captured argument (offset, size, type)
FunctionArgBlob TArray<uint8> Raw bytes of captured argument values
FunctionCallObjectPool TArray<TObjectPtr<UObject>> GC-safe objects referenced by function arguments

On resume, the VM restores these into the global context, allowing seamless continuation from inside nested function calls.

OnEnd Lifecycle Guards

Four flags manage the OnEnd event dispatch lifecycle:

Flag Type Default Description
bOnEndExecutionInProgress bool false Reentrancy guard: prevents recursive OnEnd dispatch
bOnEndEventExecuted bool false Tombstone: true once OnEnd has been attempted for this termination
bHasPendingOnEndOverride bool false True when a termination node has provided an explicit OnEnd policy
bPendingOnEndOverrideValue bool true The override value (execute OnEnd = true / skip = false)

These guards ensure that:

  1. OnEnd fires at most once per lifecycle termination
  2. A termination node (e.g., OP_End with explicit flag) can override the natural OnEnd policy
  3. Reentrancy is prevented if OnEnd logic itself triggers termination

Initialization: Context → Instance Migration

InitializeFromContext() migrates runtime state from the transient FForgeVMContext to persistent instance storage. This is the most complex operation on FStatusInstance.

Migration Steps

Structure 
InitializeFromContext(Context, StatusID)
  │
  ├─ 1. HOT DATA: Set defaults
  │      StatusID, State=Active, timers zeroed, StackCount=1
  │
  ├─ 2. COLD DATA: Copy references
  │      StatusTag, Definition, Caster/Target actors & components, Revision=1
  │
  ├─ 3. InstanceBlob: Allocate + gate state staging
  │      ├─ SetNumZeroed(InstanceBlobSize)
  │      └─ Memcpy GateStateRegionBytes from ContextStack → InstanceBlob
  │
  ├─ 4. VariableBlob: Deep copy with object remapping
  │      ├─ Bulk memcpy (handles all POD fields)
  │      ├─ For each struct variable with ComplexFields:
  │      │    └─ CopyStructFast() with Context→Instance object remapping
  │      └─ For each Object/Class single variable:
  │           └─ Read context pool index → resolve UObject* → AllocObject() → write instance index
  │
  ├─ 5. ArrayPool: Deep copy with object remapping
  │      ├─ Copy only variable slots [0, ArraySlotCount)
  │      ├─ Bulk memcpy per slot data
  │      └─ For Object/Class arrays: remap each element index
  │
  ├─ 6. Function Call State: Copy verbatim
  │      FunctionCallStack, FunctionArgSlots, FunctionArgBlob, FunctionCallObjectPool
  │
  └─ 7. Reset lifecycle guards
         bOnEndExecutionInProgress = false
         bOnEndEventExecuted = false

Object Index Remapping

The critical operation during migration is remapping object indices from the context pool to the instance pool:

Structure 
Context execution:
  ContextObjectStack[3] = SomeUObject*
  VariableBlob byte offset 42 stores: uint16(3)  ← context pool index

Migration:
  UObject* Obj = Context.GetContextObject(3)      ← resolve from context
  uint16 NewIdx = Instance.AllocObject(Obj)        ← register in instance pool
  InstanceVariableBlob byte offset 42 stores: NewIdx  ← instance pool index

After migration:
  ContextObjectStack is destroyed (reused for next execution)
  InstanceObjectPool[NewIdx] = SomeUObject*        ← GC-safe, persists

This remapping is necessary because the context object pool is transient (reused for every execution), while the instance object pool persists across ticks.

Timing Model

UpdateTiming

Called every frame for all active/suspended instances:

C++ 
void FStatusInstance::UpdateTiming(float DeltaTime)
{
    if (State == Active || State == Suspended)
    {
        ElapsedTime += DeltaTime;       // Monotonic lifetime
        TimeSinceLastTick += DeltaTime; // Reset on each tick

        if (State == Suspended)
        {
            SuspendTimer -= DeltaTime;  // Countdown to resume
        }
    }
}

Performance: All fields accessed are in cache line 1 — no cache miss during timing update.

ShouldTick

Determines if an Active instance should execute this frame:

C++ 
bool ShouldTick() const
{
    if (State != Active) return false;
    if (TickInterval > 0.f) return TimeSinceLastTick >= TickInterval;
    return true;  // TickInterval 0 = every frame
}

ShouldResume

Determines if a Suspended instance should wake up:

C++ 
bool ShouldResume() const
{
    return State == Suspended && SuspendTimer <= 0.f;
}

Duration & Refresh Semantics

Duration Timeline

Structure 
Time ──────────────────────────────────────────────────────►

Registration        Duration expires
    │                    │
    ▼                    ▼
    ├───── ElapsedTime ──┤
    │                    │
    0                 Duration (absolute expiry point)
  • ElapsedTime is monotonic — never resets, always increases
  • Duration is an absolute expiry point on the ElapsedTime timeline
  • Duration ≤ 0 means infinite (never expires)

Refresh Logic

RefreshDurationWindow() extends a finite duration without resetting elapsed time:

C++ 
void RefreshDurationWindow()
{
    if (Duration <= 0.f) return;  // Infinite: nothing to refresh

    // Recover BaseDuration if not initialized
    if (BaseDuration <= 0.f)
    {
        BaseDuration = max(Duration - ElapsedTime, Duration);
    }

    // Shift expiry forward: new absolute deadline
    Duration = ElapsedTime + BaseDuration;

    // For suspended instances with finite windows, refresh the suspend timer too
    if (State == Suspended && SuspendTimer >= 0.f)
    {
        SuspendTimer = BaseDuration;
    }
}

Example:

Structure 
BaseDuration = 10.0 seconds

At ElapsedTime = 7.0:
  Duration was 10.0 (3 seconds remaining)
  After refresh: Duration = 7.0 + 10.0 = 17.0 (10 seconds remaining again)

At ElapsedTime = 15.0:
  Duration is 17.0 (2 seconds remaining)
  After refresh: Duration = 15.0 + 10.0 = 25.0 (10 seconds remaining again)

This design ensures ElapsedTime stays monotonic (useful for analytics and UI) while allowing infinite duration refreshes via stacking policies.

Stack Management

AddStack

C++ 
void AddStack(int32 Count)
{
    StackCount += Count;
}

Simple increment. The stacking policy (Aggregate, AggregateAndRefresh, etc.) is enforced at the WorldSubsystem level before calling AddStack().

RemoveStack

C++ 
void RemoveStack(int32 Count)
{
    StackCount -= Count;
    if (StackCount <= 0)
    {
        // Auto-terminate via WorldSubsystem
        WorldSubsystem->TerminateInstance(*this, EStatusTerminateReason::StacksZero, true);
    }
}

When stacks reach zero, the instance is automatically terminated with reason StacksZero. The termination flows through the WorldSubsystem to ensure proper cleanup (OnEnd event, modifier removal, tag cleanup, etc.).

State Machine

Structure 
                    ┌────────────┐
                    │  Inactive  │ ◄── Reset() / Pool available
                    └─────┬──────┘
                          │ InitializeFromContext()
                          ▼
                    ┌────────────┐
                    │   Active   │ ◄── Executing periodically (ShouldTick)
                    └──┬─────┬──┘
                       │     │
           Handler     │     │ Handler suspends
           completes   │     │ (Delay, WaitForEvent)
                       │     ▼
                       │  ┌────────────┐
                       │  │ Suspended  │ ◄── Waiting (SuspendTimer / ForgeEvent)
                       │  └─────┬──────┘
                       │        │ ShouldResume() / Event fires
                       │        │
                       │        ▼
                       │  (Returns to Active)
                       │
                       ▼
                ┌──────────────┐
                │ Terminating  │ ◄── Duration expired / Removed / StacksZero / etc.
                └──────┬───────┘
                       │ Cleanup complete
                       ▼
                 ┌────────────┐
                 │  Inactive  │ ◄── Reset(), available for reuse
                 └────────────┘

State Descriptions

State Description Tick Behavior
Inactive Instance slot is empty / available in pool Not ticked
Pending Queued for activation (reserved for future use) Not ticked
Active Running normally. Executes on TickInterval UpdateTiming + ShouldTick check
Suspended Paused, waiting for timer or event UpdateTiming + ShouldResume check
Terminating Cleanup in progress Transitional, immediately moves to Inactive

IsActive Helper

C++ 
bool IsActive() const
{
    return State == Active || State == Suspended;
}

Both Active and Suspended are considered "alive" — they participate in timing updates, event listener registration, and termination cleanup.

Object Pool

AllocObject

Allocates or deduplicates an object reference in the instance's persistent pool:

C++ 
uint16 AllocObject(UObject* Obj)
{
    if (!Obj) return FORGE_INVALID_INDEX;  // 0xFFFF

    // Dedup: linear scan O(n), optimal for 2-20 entries
    const int32 Found = InstanceObjectPool.IndexOfByKey(Obj);
    if (Found != INDEX_NONE)
        return static_cast(Found);

    // Overflow guard: 0-65534 valid, 0xFFFF reserved
    if (InstanceObjectPool.Num() >= 0xFFFE)
    {
        // Error: pool overflow
        return FORGE_INVALID_INDEX;
    }

    return static_cast(InstanceObjectPool.Add(Obj));
}

Characteristics:

  • Deduplication — same object pointer returns same index
  • Capacity — 65,534 entries (index 0xFFFF reserved as null sentinel)
  • GC-safe — TArray<TObjectPtr<UObject>> is tracked by Unreal's garbage collector

Reset & Pool Reuse

When an instance is released back to the pool, Reset() clears all state:

C++ 
void Reset()
{
    // HOT DATA — zero all timing and control fields
    StatusID = 0;
    State = Inactive;
    ElapsedTime = 0.f;
    TimeSinceLastTick = 0.f;
    Duration = -1.f;
    BaseDuration = -1.f;
    TickInterval = 0.f;
    NextExecutionTime = 0.f;
    SuspendTimer = 0.f;
    MaxIterations = -1;
    CurrentIterationCount = 0;
    StackCount = 1;
    SavedPC = 0;
    bWaitingForAsyncEvents = false;

    // COLD DATA — null all references, free memory
    StatusTag = FGameplayTag();
    Definition = nullptr;
    CasterActor.Reset();
    TargetActor.Reset();
    CasterComponent.Reset();
    TargetComponent.Reset();
    Revision = 1;
    InstanceBlob.Empty();
    InstanceObjectPool.Reset();
    VariableBlob.Empty();
    ArrayPool.Reset();
    ActiveCustomLogic = nullptr;
    FunctionCallStack.Reset();
    FunctionArgSlots.Reset();
    FunctionArgBlob.Reset();
    FunctionCallObjectPool.Reset();
    bOnEndExecutionInProgress = false;
    bOnEndEventExecuted = false;
    bHasPendingOnEndOverride = false;
    bPendingOnEndOverrideValue = true;
}

After Reset(), the instance slot is available for reuse without reallocation. The pool manages a free list of reset instances.

Best Practices

1. Trust the Cache Line Layout

Never reorder fields in FStatusInstance. The hot/cold split is enforced by static_assert and verified at compile time. Adding new per-tick fields should go before the StatusTag boundary (offset 64).

2. Understand the Migration Cost

InitializeFromContext() is the most expensive operation on an instance due to object remapping. It runs once per instance creation, not per tick. The cost is proportional to the number of Object/Class variables and struct complex fields.

3. Use TWeakObjectPtr for Safety

Actor/component pointers are weak by design. Always check IsValid() before dereferencing in game code. The termination system handles null pointers gracefully via OwnerDestroyed cleanup.

4. Revision for Update Ordering

The Revision field provides monotonic ordering for replicated status updates. Clients can reject stale updates by comparing revision numbers, ensuring eventual consistency without locks.

5. Duration vs BaseDuration

  • Duration is the absolute expiry point — compare against ElapsedTime
  • BaseDuration is the original window size — used by RefreshDurationWindow() to compute the new absolute deadline
  • Both ≤ 0 means infinite: no expiration

6. StackCount Invariant

StackCount reaching zero triggers automatic termination via TerminateInstance(). If you need to set stacks to zero without termination, use direct field assignment followed by your own lifecycle logic — but this is strongly discouraged in normal usage.
