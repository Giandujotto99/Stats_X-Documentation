Installation

In this section you will install the plugin, enable it in your project, and create the runtime data asset that Stats_X needs to resolve compiled statuses.

Important requirement

Stats_X runtime expects a PDA_ImmutableGameStatuses data asset to be assigned in Project Settings. Without it, the plugin cannot resolve status definitions at runtime.

Install the Plugin in the Engine Version You Want to Use

First, place the plugin inside the Unreal Engine version you want to work with.

Enable Stats_X in Your Project

Open the Plugins window

In Unreal, go to Edit -> Plugins.

Search for Stats_X and enable it

Find the Stats_X plugin, enable it, and restart the editor if Unreal asks for it.

Create the Immutable Game Statuses Data Asset

Create a new Data Asset in your project content

In the Content Browser, create a new Data Asset and choose PDA_ImmutableGameStatuses as the asset class.

Give it a clear name

I recommend naming it DA_ImmutableGameStatuses. It can live anywhere in your project content folder, as long as you assign it in the settings afterward.

Assign it in Project Settings

Go to Project Settings -> Plugins -> Stats_X Runtime and assign your DA_ImmutableGameStatuses asset to Immutable Game Statuses.

Why this asset matters

Every compiled status definition is resolved through this registry asset. Later, when you compile new Status Forge assets, Stats_X will register them there automatically.

You are done: the plugin is installed correctly and the project is ready for the next setup steps.
