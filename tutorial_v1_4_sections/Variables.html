Variables

Variables let a status store values it can read inside the graph. The simplest useful example is one float variable that represents how much damage the status should deal.

What we are building

In this tutorial we create a single variable called StatsX.Variable.DamageAmount, read it in the graph, and use it as the damage value of the status.

Create the Variable

Open your Status Forge asset

Go to the Variables panel inside the status you want to use for damage.

Add one variable declaration

Create a variable with tag StatsX.Variable.DamageAmount.

Set the type and default value

Set Variable Type to Float, Container Mode to Single, and give it a default value such as 50.0.

Variable Tag Type Container Default Meaning
StatsX.Variable.DamageAmount Float Single 50.0 The amount of damage this status should apply.
Why this is already useful

Even with just one variable, the same status becomes reusable. You can keep one generic damage status and change only the value instead of duplicating the whole asset.

Use the Variable in the Graph

Read the value with Get Variable

Inside the graph, place a Get Variable node and select StatsX.Variable.DamageAmount.

Use that value as the damage input

Connect the result of Get Variable to the node that applies damage, modifies health, or feeds your custom logic.

Let the graph read the current value every time

Now the status no longer has a hardcoded damage number in the graph. It reads the value from the variable, so you can tune it from the declaration or override it from Blueprint later.

Simple Graph Flow 
Status starts
-> Get Variable (StatsX.Variable.DamageAmount)
-> Apply damage using that float value

Why This Is Better Than Hardcoding

  • You change the default damage in one place instead of rewriting the graph.
  • The same status can be reused for weak, medium, or strong versions.
  • Each cast can keep its own value independently.
Per-instance behavior

Variables belong to the running status instance. If two actors cast the same status, each one can have its own DamageAmount value.

Optionally Override It from Blueprint

If you want, Blueprint can replace the default value when the status is cast.

Create a float override

Use MakeFloatOverride with tag StatsX.Variable.DamageAmount and the new value you want, for example 125.0.

Put it into the overrides array

Add that override to the TArray<FForgeVariableOverride> you pass when casting the status.

Cast the status normally

Feed the overrides array into CastStatusAsync or CastStatusSync. The graph will read the overridden value from the very first instruction.

Blueprint Flow 
MakeFloatOverride
-> VariableTag = StatsX.Variable.DamageAmount
-> Value = 125.0

Add to Overrides Array
-> CastStatusAsync / CastStatusSync
Note

If you use the node CastStatus, the one in the Status Forge, input pins will apear based on the variables in that status. So you can override variables from Status to Status and as told from Blueprints to Status.

You are done: you now know the simplest and most useful way to start working with Variables in Status Forge.
