Thresholds

In this section you will add your first attribute threshold, bind the threshold dispatcher, and prepare a clean place where you can react to events such as death, low health, or any other critical attribute transition.

What is a threshold?

A threshold watches one attribute value and fires an event when that value reaches or crosses a specific boundary. In practice, it is the easiest way to react to events like Health.Current reaching zero.

Before you continue

This tutorial assumes your attributes are already initialized, especially Health. If you have not completed the previous section yet, do that first.

Create the Threshold Initialization Event

Open your character stats blueprint

Open BPC_CharacterStats, the same blueprint where you initialized your attributes.

Create a server event for thresholds

Add a custom event named Server_InitializeThresholds and set its replication to Run On Server.

Add the threshold registration node

As the first node in this event, add AddAttributeThreshold. From its Threshold struct input, drag a wire and create Make Attribute Threshold.

Configure the Example Threshold

This example creates a threshold for Health.Current and triggers when the value crosses below 0.

Parameter Name Parameter Value Meaning
AttributeTag Health The attribute you want to monitor.
SubAttributeTag Current We are watching the current health value, not Max or Base.
ThresholdValue 0 The boundary value that will be checked.
Comparison Crossing Below It triggers only when the value crosses from above 0 to 0 or lower.
RemoveAfterTrigger False The threshold stays active after triggering.
Example setup for a Health threshold that reacts when Current crosses below zero.

Call the Event and Bind the Dispatcher

Go back to Begin Play

In the Begin Play logic, after the call to Server_InitializeAttributes, call Server_InitializeThresholds.

Bind the threshold dispatcher

Next to that logic, add BindEventToOnAttributeThresholdReached.

Create the matching event

From the red event input pin, create a Create Event node and choose Create a matching event from the signature dropdown.

Name the callback clearly

Rename the generated event to OnAttributeThresholdReached_ServerEvent so it is obvious where this logic runs and what it reacts to.

Bind the dispatcher so the component can react when a threshold is crossed.

What to Do Inside the Event

Your threshold setup is now ready. Inside OnAttributeThresholdReached_ServerEvent, you can decide what should happen when the threshold is triggered.

  • Filter by AttributeTag if you only care about a specific monitored attribute.
  • Filter by threshold handle if you are storing handles in a map or array and want more precise control.
  • Use this event for reactions such as death, phase changes, warnings, or automatic gameplay responses.
Persistent vs one-shot thresholds

Because RemoveAfterTrigger is set to False in this example, the threshold remains active after firing. If you need a one-time trigger, enable removal after trigger instead.

You are done: thresholds are now registered, bound, and ready to drive gameplay logic from attribute changes.
